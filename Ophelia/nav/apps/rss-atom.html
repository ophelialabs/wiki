What Are RSS/Atom Feeds?

RSS and Atom are XML-based formats for publishing frequently updated content such as blogs, news, or project updates. Many sites expose their latest content via a feed URL (e.g., https://example.com/feed.xml or https://blog.example.com/rss).

How to Find a Siteâ€™s Feed

  • Look for the RSS icon 📱 or links labeled "RSS" or "Atom".
  • Common feed URLs: /feed, /rss, /atom.xml, /feed.xml
  • Check the HTML <head> for: 
    <link rel="alternate" type="application/rss+xml" href="https://example.com/feed.xml" />

Fetching and Parsing RSS/Atom Feeds in JavaScript

Note: Browsers block cross-origin requests to most RSS feeds (CORS). Use a public RSS-to-JSON proxy (like rss2json.com, rss.app, or your own backend) to fetch and parse feeds.

Example: Fetch and Search RSS Feed Client-Side

const feedUrl = encodeURIComponent('https://blog.example.com/feed.xml');
fetch(`https://api.rss2json.com/v1/api.json?rss_url=${feedUrl}`)
  .then(res => res.json())
  .then(data => {
    // data.items is an array of feed entries
    const results = data.items.filter(item =>
      item.title.toLowerCase().includes(query.toLowerCase()) ||
      item.description.toLowerCase().includes(query.toLowerCase())

Best Practices & Notes

  • Respect API limits of RSS-to-JSON proxies.
  • For production, consider running your own backend proxy to fetch and parse feeds server-side.
  • Always credit the source of each result.

Summary

  1. Find the RSS/Atom feed URL for the site(s) you want to federate.
  2. Use a proxy to fetch and parse the feed as JSON.
  3. Search the entries client-side and merge with your own results.
  4. Display all results, clearly labeled by source.
