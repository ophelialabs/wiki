Flask is a Python project that demonstrates how to build web applications using Flask and Jinja2.

Best Use Cases

  • Learning Flask and Jinja2: Straightforward, well-documented codebase for beginners to grasp web development fundamentals in Python.
  • Prototyping Web Applications: Minimal setup lets you quickly spin up working web apps for testing ideas or proof-of-concept projects.
  • Small to Medium Projects: Modular structure suitable for personal projects, internal tools, or MVPs, allowing easy growth.
  • Template Customization: Jinja2 enables reusable components, custom filters, and clean separation of logic and presentation.
  • Educational and Teaching Tool: Practical resource for instructors and students in workshops, tutorials, or classrooms.

This project provides a solid foundation for building web applications that require dynamic content, user interaction, and maintainable code.

Example Use Cases & Related Technologies

  • Personal Portfolio Website: Dynamic portfolio with routes for projects and blog posts, rendered from Python data or a database.
  • Internal Dashboards: Team dashboards for analytics or server status, integrating APIs and interactive charts (e.g., Chart.js).
  • Event Registration Forms: Registration pages with form handling and feedback using Flask and Jinja2.
  • API Prototyping: Quickly prototype RESTful APIs and test behaviors before full-scale development.
  • Educational Demos: Demonstrate authentication, template inheritance, or form validation in a classroom setting.
Flask Extensions
  • Flask-WTF: Form creation & validation
  • Flask-Login: User authentication
  • Flask-SQLAlchemy: SQL database integration
Front-End Libraries
  • Bootstrap, Tailwind CSS: Styling & responsiveness
  • Alpine.js, Vanilla JS: Interactivity
Testing Tools
  • pytest, unittest: Automated route & logic testing
Deployment Platforms
  • Heroku, PythonAnywhere, Docker: Cloud/container deployment
Database Backends
  • SQLite: Lightweight, file-based
  • PostgreSQL, MySQL: Production-ready, scalable

Features

  • Lightweight Flask web server
  • Jinja2 templating for dynamic HTML
  • Easy to extend and customize
  • Live Reload

Getting Started

Reference: Authlib OAuth2 Example

  1. Clone the repository: 
    git clone https://github.com/jlabclouds/Flask.git
cd Flask
  2. Install dependencies: 
    pip install -r requirements.txt
  3. Set Flask and Authlib environment variables 
    # DO NOT SET THIS IN PRODUCTION!
export AUTHLIB_INSECURE_TRANSPORT=1
  4. Create Database and Run the application: 
    python app.py
flask app __init__.py
  5. Open your browser:
    Visit http://localhost:5000
  6. Access the API: 
    curl http://localhost:5000/api

    Check the API response in your terminal.

  7. From Scratch 
    mkdir my_flask_app
cd ~/my_flask_app
python -m venv .venv
source .venv/bin/activate
# Windows
# .venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install PyQt6 PyQtWebEngine
pip install -U huggingface_hub[cli,torch,tensorflow]
pip install --upgrade tensorflow
# Current stable release for GPU (Linux / WSL2)
pip install tensorflow[and-cuda]

# Create requirements.txt
cat <<EOF | sudo tee /my_flask_app/requirements.txt
Flask>=3.0.3
Flask-SQLAlchemy
Authlib
EOF

# Create main.py, runserver.py
cat <<EOF | sudo tee /my_flask_app/main.py
import os
import flask
EOF

python -m pip install -r requirements.txt
  8. Passwd flow example
    • Get client_id and client_secret
    curl -u ${client_id}:${client_secret} -XPOST http://127.0.0.1:5000/oauth/token -F grant_type=password -F username=${username} -F password=valid -F scope=profile
    • Note password=valid
    curl -H "Authorization: Bearer ${access_token}" http://127.0.0.1:5000/api/me
    • Grant Auth, send code to Auth server to get access token
    curl -u ${client_id}:${client_secret} -XPOST http://127.0.0.1:5000/oauth/token -F grant_type=authorization_code -F scope=profile -F code=${code}
    curl -H "Authorization: Bearer ${access_token}" http://127.0.0.1:5000/api/me
  9. Auth flow example
    • Test the auth code flow
    open http://127.0.0.1:5000/oauth/authorize?response_type=code&client_id=${client_id}&scope=profile
